Stop wasting time navigating menus, managing hundreds of identical browser tabs, copying IDs from the URL, and clearing cookies. Domo Toolkit is built for the people who live inside Domo every day: administrators managing hundreds of objects, consultants jumping between client instances, and power users building on the platform. Every feature saves clicks, reduces context-switching, and surfaces information Domo's native UI buries, putting the tools you need one click away.

INSTANT CONTEXT DETECTION
Domo Toolkit automatically detects what you're looking at. Navigate to any page, card, dataset, dataflow, app studio page, workflow, alert, or any of many supported object types, and the extension identifies it, enriches it with metadata from the Domo API, and makes relevant actions available. It even detects modals without URLs, resolves parent objects for nested types, detects selected code engine and form actions inside workflow editors, and updates your tab title with the object's name. With one click you can explore the object's full JSON definition, with epoch timestamps shown as readable dates and user/group IDs as display names. Most objects also expose related data the same way, like a card's datasets or a dataset's columns.

AUTOMATIC 431 ERROR RESOLVING
Domo's 431 "Request Header Fields Too Large" errors are a constant headache when working across instances. Domo Toolkit gives you three independent cookie controls:
- Auto-clear on 431 errors (default on): Detects 431 errors, clears cookies (preserving your last 2 instances), and refreshes the page instantly.
- Manual cookie button (default on): A one-click toolbar button to clear without waiting for a 431.
- Button behavior (default preserve): Preserve your last 2 instances or clear all Domo cookies, leaving other websites and your history untouched either way.

ONE-CLICK ACTIONS
A compact toolbar gives instant access to the operations you use most:
- Copy ID: Copy the current object's ID instantly. Long-press for related IDs like stream ID (datasets) or app ID (app studio pages). Shortcut: Ctrl+Shift+1 (Cmd on Mac).
- Share With Self: Instantly grant yourself access to pages, studio apps, and custom app designs.
- Activity Log: View activity log records for the current object. Long-press for advanced scopes: all cards on the object, all pages containing those cards, all child pages, or just the parent Studio App / Worksheet. Supports multi-user filtering with an include/exclude toggle, and can pull from a DomoStats Activity Log dataset to bypass the audit API's retention limit.
- Clipboard Navigation: Read your clipboard, identify the Domo object by its ID, and navigate straight to it. Works with any Domo object ID copied from anywhere: a card, spreadsheet, Slack message, etc. Objects that don't support navigation show details in the side panel instead.
- Delete Current Object: Delete beast modes, appdb collections, workflows, pages/app studio pages and all their cards, dataflows and all their outputs, and approval templates. Lists dependencies before you confirm, split into "Will be deleted" and "Other". Pages with child pages are blocked from deletion.

DATA DISCOVERY
View dependencies and take actions on them from the side panel:
- Get Cards: See every card on a page, app studio page, worksheet page, report builder page, dataset, or dataflow outputs. For pages, forms and queues are included, separated from cards. Open them all in new tabs at once.
- Get Card Pages: Shows where cards on an object live. For pages, shows other pages containing the same cards. For datasets and dataflows, shows all pages and app studio pages with cards that use them as a source.
- Get Child Pages: For pages, shows all child and grandchild pages in a tree.
- Get DataSets: Shows datasets for an object, including dataflow inputs and outputs, dependent views, and all datasets used in cards on a page or app.
- Get DataSets Used in View: For dataset views and datafusions, see the source datasets that feed the view.
All discovery lists support applicable actions like open all, copy ID, share with self, open lineage, and open in views explorer. Items are grouped with expand/collapse, counts, direct links, and icons, and a reload button reruns the action as you navigate.

API ERROR TRACKING
API errors are automatically captured as you browse. Click the API Errors button to view the count at a glance, expand individual errors for the full JSON response, and clear all with one click.

USER MANAGEMENT
- Transfer Ownership: Reassign every object owned by a user to another in one flow. Select which object types to include, preview counts before you commit, and optionally delete the user afterward. A quick button transfers to the user's manager, and you can email the recipient an Excel attachment listing everything transferred.
- View Ownership by User: See everything a user owns, grouped by object type. A virtualized list keeps large results fast.
- Duplicate (Clone) User: Clone a user with all access, group membership, and configuration carried over; just change the name and email. Choose which individually-shared cards, pages, and apps to re-share with per-item checkboxes, and get an Excel audit log of every item and its result.

OBJECT-SPECIFIC ACTIONS
- Lineage: Open a full-page lineage graph for datasets and dataflows, tracing upstream and downstream dependencies with dataset previews and dataflow tile operations. Dark mode supported.
- Copy Filters: Copy a URL with all applied filters on a card, page, or app studio page.
- Export Data: Export card data as CSV or Excel, including applied filters (even from a card modal), and export code engine package versions as JavaScript/Python files.
- Data Repair: Open the hidden data repair tab for any dataset.
- Update Owner: Change the owner of alerts and workflows with a searchable picker and a "Set to Self" button.
- Update DataFlow Details: Edit a dataflow's name and description without a new version.
- Update DataSet Details: Update a dataset's user-defined type.
- Migrate DataSet Content: Repoint every beast mode, card, drill path, dataflow, and dataset view that uses a dataset to a different one in a single pass. Deselect a whole type or cherry-pick items, run a schema check that flags missing or mismatched columns, and remap them (including in formulas and SQL) before applying.
- Sync Datastore: Trigger a sync on an AppDB collection.
- Generate Schema: Infer a column schema for an AppDB collection from its recent documents, edit it, and apply it (optionally turning on sync to produce a dataset).
- Update Code Engine Versions: Bulk update workflow code engine actions to the latest version without unmapping inputs and outputs.
- Copy Color Rules: Copy a dataset's color rules to another.
- Cancel Stuck Stream Update: Cancel a dataset stream execution stuck in a storing state with no UI cancel option.
- Generate Definition from JSDoc: Derive a Code Engine package manifest (function names, parameters, types, descriptions) from JSDoc in a package's source and update the definition to match, with a structural diff before you confirm.
- Lock Cards: Lock all cards on a page, app studio page, dataset, worksheet, report builder report, or all dataflow outputs at once.
- Set Schedule to Manual: Set a dataset's update schedule to manual.
- Fix Empty String Filters: Remove empty string default values from "contains" quick filters on cards, so null values display instead of being filtered out.

CUSTOM FAVICONS FOR MULTI-INSTANCE MANAGEMENT
Across multiple Domo instances, every tab looks identical: same favicon, same "Domo" title. Domo Toolkit sets the tab title to the object and lets you customize favicons with regex rules:
- Instance Logo: Use the instance's own logo as the favicon.
- Colored Domo Logo: Change the background of the Domo logo with a custom color.
- Colored Stripes: Add a colored stripe to the top, right, bottom, or left edge.
- Regex Patterns: Match instance subdomains with flexible patterns.
- Priority Ordering: Drag and drop to set rule priority.

SIDE PANEL & POPUP
Access Domo Toolkit in two ways:
- Popup: Click the extension icon; there when you need it, gone when you don't.
- Side Panel: A persistent panel alongside your page for data discovery, with collapsible action buttons so it stays compact. Opens automatically from the popup to display discovery features as a list.
Both show the current context (instance, object type, object ID) and update automatically as you navigate.

SETTINGS
- Theme: System, light, or dark mode.
- Default Domo Instance: Set your go-to instance for clipboard navigation from non-Domo sites.
- Cookie Clearing: Three toggles for auto-clearing on 431 errors, the manual button, and its behavior.
- Favicon Rules: Full editor with pattern matching, effect selection, color picker, and drag-and-drop reordering.
- Per-Instance Settings: View, manage, and clear values stored per Domo instance, like the auto-discovered DomoStats Activity Log dataset.

SUPPORTED OBJECT TYPES
Domo Toolkit works with nearly every object type in Domo: pages, cards, datasets, dataflows, app studio apps/pages, worksheets, workspaces, report builder, users, groups, alerts, workflows, code engine packages, pro-code apps, beast modes, variables, access tokens, appdb collections, approvals, approval templates, drill paths, jupyter workspaces, filesets, files, forms, governance toolkit jobs, certification processes, AI toolkits, AI agents, and dozens more.

PRIVACY & PERMISSIONS
- Only runs on .domo.com domains
- Uses Domo's existing authenticated session, no additional login required
- No data leaves the browser; no external servers are contacted
- Settings sync via Chrome's built-in storage
- Domo data is never read, stored, or sent off-device
